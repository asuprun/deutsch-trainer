# 01 — Обзор

## Идея проекта

Личное PWA для изучения немецкого языка с переводом на русский. Пользователь делает скриншот страницы PDF-учебника (или фото, или копирует картинку из интернета), загружает в приложение — AI разбирает изображение, извлекает слова, грамматические правила и примеры, переводит на русский, и автоматически формирует флешкарты для интервального повторения по алгоритму FSRS (как в Anki).

## Цели (в scope для v1)

- Загрузка скриншота страницы немецкого учебника (или любой фото с немецким текстом).
- AI извлекает: слова с переводом, устойчивые фразы, грамматические правила, примеры из текста.
- Автоматическое формирование флешкарт с метаданными (род, формы глагола, тип слова).
- Тренировка по алгоритму FSRS-5.
- Озвучка немецких слов (TTS).
- Дашборд с прогрессом (heatmap активности, streak, retention).
- Работа с телефона как нативное приложение (PWA с установкой на главный экран).
- Доступ снаружи (любая сеть) и полностью бесплатное на free-tier хостингах.

## Не-цели (вне scope для v1)

- Многопользовательский режим, регистрация, шаринг колод между людьми.
- Нативные iOS/Android приложения (отдельные сборки).
- Социальные функции — друзья, leaderboard, обмен прогрессом.
- Генеративные диалоги с AI (чат-практика, ролевые игры) — возможно в v2.
- Импорт из Anki/Quizlet — экспорт в Anki будет, импорт пока нет.
- Распознавание рукописного текста с высокой точностью — это бонус, не гарантия.

## Обоснование ключевых выборов

### Почему Gemini 2.0 Flash, а не Claude API или OpenAI

- **Бесплатный тир щедрый:** 15 RPM, 1500 запросов в день, 1M tokens per minute — этого хватит для личного использования с большим запасом.
- **Vision из коробки:** принимает изображения в одном запросе с текстом — не нужно отдельное OCR-решение.
- **Structured Output:** поддержка `responseSchema` (JSON Schema) гарантирует валидный JSON — критично для автоматизации.
- **Хороший немецкий и русский:** Google-модели сильны в multilingual задачах.
- **Скорость:** Flash отвечает за 1–3 секунды.

Резервный вариант — Groq (Llama 3.2 Vision) с переключением через env-флаг.

### Почему Supabase, а не Cloudflare D1 или Firebase

- **Postgres, а не SQLite:** JSONB, GIN-индексы для тегов, нативные timestamps — полезно для FSRS-полей.
- **Auth, Storage, DB в одной консоли:** меньше движущихся частей.
- **Бесплатный тир:** 500MB БД + 1GB Storage + 2GB трафика — реально хватает.
- **TypeScript-клиент с автоген типами** из схемы БД.

### Почему Vercel, а не Netlify или Cloudflare Pages

- **Лучшая интеграция с Next.js** — оба продукта одной компании.
- **Edge runtime для API routes** — быстрее, чем serverless cold start.
- **Хорошие лимиты Hobby:** 100GB трафика, 1M edge function invocations.

### Почему FSRS, а не SM-2 (классический Anki) или простые интервалы

- **FSRS-5** научно обоснован, учитывает стабильность памяти и сложность карты.
- В среднем требует **на 30% меньше повторений** при той же ретенции, что SM-2.
- Реализация `ts-fsrs` — официальная JS-библиотека от автора алгоритма, ~30KB.

### Почему dark mode по умолчанию

- Прямой запрос пользователя.
- Лучше для длительных вечерних сессий.
- Тёмная тема — стандарт для современных продуктивных приложений.
